# أزرار — إدارة الخياطة الرجالية الحديثة 🧵

تطبيق أندرويد كامل (Capacitor + JavaScript) لإدارة محل خياطة رجالية — يعمل **دون اتصال بالإنترنت** وجاهز للبناء التلقائي عبر GitHub Actions.

---

## ✨ المميزات

- **لوحة تحكم** محمية بكلمة مرور للإحصائيات المالية (الافتراضية: `0000`) مع تنبيهات ذكية (مبالغ متبقية، اقتراب/تأخر مواعيد التسليم، مستحقات الموردين، الثوب المجاني).
- **العملاء**: حفظ آخر القياسات واسترجاعها تلقائياً، زر جهات الاتصال، عداد الأثواب (بعد 9 أثواب العاشر مجاني 🎁)، نسخ آخر فاتورة.
- **الفواتير**: رقم تلقائي، تاريخ ويوم تلقائي، موعد تسليم، نوع تفصيل (سعودي/قطري/إماراتي/أخرى)، دفع بثلاث عملات (ريال سعودي/يمني/دولار) بأسعار صرف قابلة للتعديل (التوحيد بالريال اليمني)، 7 قياسات مع كسور (1/4، 1/2، 3/4)، تفاصيل قصات كاملة (رقبة قلاب/سادة، جيب، أكمام، كبك، جبزور)، صورة قماش، ملاحظات.
- **الفاتورة النهائية**: شعار المحل، الحسابات البنكية الثابتة بعنوان «نفخر لثقتكم بنا»، رموز QR واتساب وإنستغرام (تُحدَّث من الإعدادات).
- **إعدادات الطباعة**: حجم الورق A4 / A5 / A6، اتجاه عمودي/أفقي، طباعة الفاتورة كاملة أو **القياسات فقط**، معاينة مع تكبير/تصغير، ثم: طباعة مباشرة / PDF / واتساب / بلوتوث / Wi-Fi.
- **الموردون**: سجل شراء ودفعات مع حساب المستحقات تلقائياً.
- **الحسابات والتقارير**: إيرادات، مصروفات، صافي أرباح، كشف حساب برصيد جارٍ، تقارير يومية/أسبوعية/شهرية/سنوية مع تصدير PDF.
- **نسخ احتياطي** كامل (تصدير/استعادة ملف JSON) لنقل البيانات بين الأجهزة.

## 🚀 خطوات الرفع والبناء (بدون Android Studio)

### 1) إنشاء المستودع ورفع الملفات
1. أنشئ مستودعاً جديداً على GitHub (Public أو Private).
2. ارفع كل محتويات هذا المجلد عبر **Add file → Upload files**.

### 2) ⚠️ مهم: إنشاء ملف الـ Workflow يدوياً
رفع المتصفح **يتجاهل المجلد المخفي `.github`**، لذلك بعد رفع الملفات:

1. من صفحة المستودع اضغط **Add file → Create new file**.
2. في خانة اسم الملف اكتب حرفياً:
   ```
   .github/workflows/build-apk.yml
   ```
   (كتابة `/` تنشئ المجلدات تلقائياً)
3. انسخ محتوى الملف **`workflow-build-apk.yml`** الموجود في جذر المشروع والصقه.
4. اضغط **Commit changes**.

### 3) تنزيل التطبيق
1. سيبدأ البناء تلقائياً — تابعه من تبويب **Actions** (يستغرق 5–8 دقائق).
2. افتح آخر تشغيل ناجح ✅ ثم من قسم **Artifacts** نزّل **`azar-tailoring-apk`**.
3. فك الضغط وثبّت `azar-tailoring.apk` على الجوال.

## ☁️ المزامنة بين جوالين (تلقائية وفورية)

يستخدم التطبيق **Supabase** (مجاني) للمزامنة. أي تعديل من أي جوال يظهر في الجوال الآخر فوراً عند توفر الإنترنت، ويعمل التطبيق أوفلاين ويرفع التغييرات تلقائياً عند عودة الاتصال.

### 1) إنشاء المشروع (مرة واحدة فقط)
1. سجّل مجاناً في **supabase.com** ثم اضغط **New Project** (اختر أي اسم وكلمة مرور لقاعدة البيانات).
2. بعد إنشاء المشروع افتح من القائمة الجانبية **SQL Editor** → **New query** والصق الكود التالي ثم اضغط **Run**:

```sql
create table if not exists shop_data (
  shop_id    text not null,
  rec_type   text not null,
  rec_id     text not null,
  data       jsonb not null,
  updated_at bigint not null,
  deleted    boolean default false,
  primary key (shop_id, rec_type, rec_id)
);

alter table shop_data enable row level security;

create policy "open_access" on shop_data
  for all using (true) with check (true);

alter publication supabase_realtime add table shop_data;
```

3. من **Project Settings → API** انسخ:
   - **Project URL** (مثل `https://xxxxx.supabase.co`)
   - **anon public** key

### 2) تفعيل المزامنة في التطبيق (في كلا الجوالين)
افتح **الإعدادات → المزامنة عبر الإنترنت** وأدخل:
- رابط المشروع + مفتاح anon
- **معرّف المحل**: أي كلمة تختارها — **يجب أن تكون نفسها في الجوالين** (مثل `azrar-shop-1`)
- اختر «مفعّلة» ثم **حفظ وتشغيل المزامنة**

الجوال الأول يرفع بياناته كاملة، والجوال الثاني يستقبلها تلقائياً. بعدها كل تعديل (فاتورة، عميل، دفعة، إعدادات...) يتزامن فوراً، والنقطة الملونة بجانب اسم التطبيق تبين الحالة: 🟢 تعمل | 🟠 جارٍ الرفع | 🔴 تعذر الاتصال.

> ⚠️ **ملاحظة أمنية**: احتفظ بمفتاح anon ومعرّف المحل لنفسك — أي شخص يعرفهما يستطيع الوصول للبيانات. لا تشاركهما خارج جوالاتك.

## 📝 ملاحظات

- **الطباعة المباشرة داخل التطبيق**: تفتح قائمة المشاركة لاختيار خدمة الطباعة أو تطبيق الطابعة (هذه طريقة أندرويد المعتمدة داخل التطبيقات). في المتصفح تعمل الطباعة مباشرة.
- **كلمة المرور الافتراضية**: `0000` — غيّرها من الإعدادات.
- **أسعار الصرف**: من الإعدادات (1 دولار = ر.ي، 1 ريال سعودي = ر.ي) وتُستخدم لتوحيد الإحصائيات بالريال اليمني.
- جميع البيانات محفوظة محلياً على الجهاز — احرص على **النسخ الاحتياطي** بشكل دوري.

```
├── workflow-build-apk.yml     نسخة ملف البناء (للصقها يدوياً — الخطوة 2)
├── .github/workflows/         ملف البناء التلقائي

└── www/                       التطبيق (HTML/CSS/JS + مكتبات PDF محلية + الشعار)
```
